## Added

- **Contrast semantic tokens** (`theme/tokens.less` + `theme/dark.less`):
  - `--ran-color-contrast-bg` â†’ `var(--ran-gray-1000)` (`#171717` light / `#ededed` dark)
  - `--ran-color-contrast-bg-hover` â†’ `#383838` light / `#cccccc` dark
  - `--ran-color-contrast-bg-active` â†’ `#4d4d4d` light / `#b3b3b3` dark
  - `--ran-color-contrast-text` â†’ `var(--ran-background-100)` (`#fff` light / `#000` dark)

  bg/text ride the gray/background scales so they invert automatically; the
  hover/active steps have no scale equivalent, so they are literals overridden
  in the dark mixin (same precedent as `--ran-color-bg-elevated`).

- **`r-button type="contrast"`** â€” a first-class variant consuming the tokens
  (default / hover / active / ripple), mirroring the `[type="primary"]` blocks
  in `components/button/index.less`. This is the Geist "contrast" button:
  black-on-white in light, white-on-black in dark. JSDoc + generated docs updated.

## Changed

- **`r-card` default surface is now bordered, not filled**
  (`components/card/index.less`): background `--ran-color-bg` + 1px
  `--ran-color-border`, replacing the `--ran-color-bg-muted` gray fill. This is
  the Geist card default. Consumers wanting the old inset look can set
  `--ran-card-background: var(--ran-color-bg-muted)`.
  Also added `--ran-card-border-color` so consumers can drive hover
  border-darkening through the closed shadow root
  (`r-card:hover { --ran-card-border-color: var(--ran-color-border-hover) }`).
- **Default (secondary) button hover de-blued**: border darkens on the gray
  ladder (400 â†’ 500) and text stays `--ran-color-text`, instead of flipping to
  `--ran-color-primary`; the default ripple is now `--ran-gray-alpha-400`.
