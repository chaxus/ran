# 2026-07-11 — Geist contrast action + bordered card default

> Follow-up the same day: `<r-theme-switch>` (three-state theme pill, syncs
> instances/tabs + theme-color metas), self-hosted Geist faces as
> `ranui/fonts` (variable woff2 + OFL license, ~138 KB), and an opt-in
> `r-card hoverable` attribute (border-darken + elevated shadow) so consumers
> stop hand-rolling these three at the site layer. See CHANGELOG [Unreleased].

Driven by the Vercel/Geist-style redesign of edit.chaxus.com (a ranui consumer):
the site needed a monochrome "highest-contrast" primary action and true
Geist-style bordered cards, and per the ecosystem-first rule those primitives
belong upstream in ranui, not as site-level overrides.

## Added

- **Contrast semantic tokens** (`theme/tokens.less` + `theme/dark.less`):
  - `--ran-color-contrast-bg` → `var(--ran-gray-1000)` (`#171717` light / `#ededed` dark)
  - `--ran-color-contrast-bg-hover` → `#383838` light / `#cccccc` dark
  - `--ran-color-contrast-bg-active` → `#4d4d4d` light / `#b3b3b3` dark
  - `--ran-color-contrast-text` → `var(--ran-background-100)` (`#fff` light / `#000` dark)

  bg/text ride the gray/background scales so they invert automatically; the
  hover/active steps have no scale equivalent, so they are literals overridden
  in the dark mixin (same precedent as `--ran-color-bg-elevated`).

- **`r-button type="contrast"`** — a first-class variant consuming the tokens
  (default / hover / active / ripple), mirroring the `[type="primary"]` blocks
  in `components/button/index.less`. This is the Geist "contrast" button:
  black-on-white in light, white-on-black in dark. JSDoc + generated docs updated.

## Changed

- **`r-card` default surface is now bordered, not filled**
  (`components/card/index.less`): background `--ran-color-bg` + 1px
  `--ran-color-border`, replacing the `--ran-color-bg-muted` gray fill. This is
  the Geist card default. Consumers wanting the old inset look can set
  `--ran-card-background: var(--ran-color-bg-muted)`.
  Also added `--ran-card-border-color` so consumers can drive hover
  border-darkening through the closed shadow root
  (`r-card:hover { --ran-card-border-color: var(--ran-color-border-hover) }`).
- **Default (secondary) button hover de-blued**: border darkens on the gray
  ladder (400 → 500) and text stays `--ran-color-text`, instead of flipping to
  `--ran-color-primary`; the default ripple is now `--ran-gray-alpha-400`.

## Fixed

- Swept the last antd-era hardcoded fallbacks (`#1890ff`, `#40a9ff`,
  `#d9d9d9`) in button / card / section / input / checkbox / colorpicker /
  select / message for current Geist values (`#006bff`, `#eaeaea`, `#f2f2f2`),
  so a missing token layer degrades to the right palette.

## Verification

- `pnpm --filter ranui build` clean; `dist/ranui.css` carries the contrast
  tokens and `dist/iife/button.iife.js` the variant.
- `button.contract` / `card.contract` / `colorpicker.contract` /
  `utils.signal` suites pass (77 tests).
- Verified rendered on edit.chaxus.com dev: contrast CTA inverts across
  light/dark, cards render bordered in both themes, card hover darkens the
  border via the new component token.

## docs (ran.chaxus.com) — unified onto the Geist design model

Same-day follow-up: the docs site joins ranui + edit.chaxus.com on one design
model. Typography (self-hosted Geist) and the `--ran-*` token layer (via
`ranui/style`) were already loaded — the gap was that VitePress chrome and the
custom home still ran their own palettes.

- **`styles/vars.less`**: `--vp-c-brand*` remapped from the stock VitePress
  purple (`#646cff`) onto the `--ran-blue` ladder (`#006bff` accent);
  `--vp-c-bg/-alt/-soft/-divider/-text-*` mapped to the `--ran-background/gray`
  scales in both light and `.dark` (explicit `.dark` overrides are required to
  beat the default theme's own values); `--vp-button-brand-*` becomes the Geist
  contrast (monochrome) button, matching `r-button type="contrast"`.
- **`Home.vue`**: tri-color aurora/gradients (`#7c5cff/#4aa6ff/#ff5db1`) are
  gone — wordmark is solid ink in Geist (Space Grotesk display face dropped),
  primary CTA is a contrast pill, pillar icons are monochrome bordered chips,
  stats/headings solid ink, blue reserved for links/accents, green only for
  status. All values ride `--ran-*` tokens with literal fallbacks. The page-local
  `--ran-g1/2/3` variables (which squatted on the ranui namespace) are removed.
- **Cleanup**: navbar title is plain ink (gradient text-clip removed);
  `theme-color` meta becomes the light/dark `#ffffff`/`#000000` pair; dead
  `theme/theme.less` (old trading-app brand) and the unused display faces
  (Fraunces/Syne/Space Grotesk woff2 + @font-face) are deleted.
- Verified: home + component pages in light/dark, `pnpm tsc`, full
  `pnpm build` (sitemap, llms-full.txt, sw precache) green.
