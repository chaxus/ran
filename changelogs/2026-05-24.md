- **`test/unit/select-accessibility.test.ts`** — `r-select` 无障碍合约测试：
  - `aria-disabled` 属性传播

  - Button（多种 type）、Icon、Input、Select（关闭 / 展开下拉）、Checkbox、Progress、Loading（冻结动画）、Skeleton、Tabs（默认 / 切换标签）、Popover（关闭 / hover 触发展开）、Math/KaTeX、Radar canvas、Form 布局、Modal（关闭 / 打开状态）

  - 默认外观、面板展开状态、RGBA 颜色值渲染

- **`test/e2e/visual/player.spec.ts`** — 3 个视频播放器 UI 视觉测试：
  - 无 src 初始状态、控制栏布局、demo 页面 HLS 加载状态

#### CI 工作流

- **`.github/workflows/visual-regression.yml`** — 独立的视觉回归 GitHub Actions 工作流：
  - 触发条件：`packages/ranui/**` 有变更的 PR / push 到 main
  - 仅用 Chromium 运行以保持结果确定性
  - 传递 `ARGOS_TOKEN` secret 上传截图至 Argos CI，PR 中自动展示视觉差异报告
  - 将 Playwright HTML 报告上传为 artifact（保留 7 天）

## 使用说明

### 本地运行视觉回归测试

```bash
# 启动 dev server（如未运行）
pnpm --filter ranui dev

# 在另一个终端运行视觉测试（仅 Chromium）
pnpm --filter ranui exec playwright test --project=chromium test/e2e/visual/

# 查看截图报告
pnpm --filter ranui test:report
```

### CI 接入 Argos

1. 在 [argos-ci.com](https://argos-ci.com) 注册并关联 GitHub 仓库（开源项目免费）
2. 将获得的 token 添加至 GitHub Secrets：`Settings → Secrets → ARGOS_TOKEN`
3. 每次 PR 触发 `visual-regression.yml` 后，Argos 自动对比截图并在 PR 中评论差异报告
