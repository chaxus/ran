# Changelog — 2026-06-28

## packages/ranui

延续 [2026-06-27](./2026-06-27.md) 的主题系统重做（PR #350 已合并入 main），本日修复合并前 CI 暴露的问题，并重做 colorpicker 组件。

### r-select：下拉箭头图标自注册 + 暗色适配

- **问题**：`r-select` 渲染 `<r-icon name="arrow-down">` 作为下拉箭头，却从未注册该图标，导致箭头缺失并打印 `icon not registered` 警告——除非使用方自己注册 `arrow-down`。箭头颜色还硬编码为 `#d9d9d9`，暗色模式下不跟随主题。
- **修复**：select 在模块加载时 `registerIcon('arrow-down', …)` 自注册图标，组件自包含；箭头颜色改为 `var(--ran-color-text-secondary)`，随主题翻转。

### demo：图标注册时序

- 抽出 `demo/register-icons.ts`，作为第一个 side-effect import，确保 demo 图标在任何组件模块（尤其是会在加载时注册图标的 `r-message`）之前注册，消除路由内容挂载时的 `icon not registered` 警告。

### 可视化回归测试：对齐路由化 demo

- 为 `/components` 路由的每个展示块补充稳定的 `id="component-*"` 锚点；tabs 使用语义化 `r-key`（overview/api/theming/disabled）。

### r-colorpicker：面板重做

- **根因**：`r-popover` 会把面板内容 portal 到 `document.body`，脱离组件 shadow root，于是所有 shadow 作用域 CSS 失效——饱和度方块坍缩为 0px、滑块脱锚浮动、输入框沦为无功能装饰，且大量硬编码颜色不支持暗色。
- **重做**：
  - 面板样式移入 `panel.less`，以 `<style>` 注入面板子树，随 portal 内容一起迁移；选择器唯一命名、全部走 token，暗色自动生效（token 从 light DOM 的 `:root` 继承）。
  - 用基于百分比、与布局无关的自定义 hue/alpha 滑块替换脆弱的 `r-progress` 滑块。
  - 接通取值输入框 + HEX/RGB 格式选择，可显示与编辑颜色，并派发 `change` 事件（`{ value, hex, rgb, rgba, alpha }`）。
  - Geist 对齐的间距 / 圆角 / 预览色块 / 棋盘格（token 驱动），触发块加入 hover 态。
- 已在明暗两色验证；契约测试扩展（alpha 解析、`currentValue`、`change` 事件），视觉 e2e 用例通过。

### DX 改进（消费侧体验）

针对使用方反馈的痛点：

- **类型可发现性（TS7016）**：`ranui` 增加 `typesVersions` 兜底，使 `moduleResolution: node`（经典模式，忽略 `exports.types`）的消费方也能解析 `ranui/builder`、`ranui/ssr`、各组件子路径的声明文件。根因复盘：经典解析忽略 `exports.types`；另外 workspace 链接的依赖若 `dist` 未构建（如 `ranuts` 缺 `.d.ts`），消费方就会 TS7016——`ranuts` 重新构建后 159 个 `.d.ts` 恢复正常，属 dist 陈旧而非配置错误。
- **命令式属性别扭**：`r-select` 的 `value` 加入 `observedAttributes`，并在变更时通过 `selectOptionElement` 刷新闭合态标签——`select.value = 'x'` / `setAttribute('value','x')` 直接更新所选项，无需再 “nudge”。
- **下拉箭头未对齐**：`r-select` 的箭头图标原先用 `top:50%; margin-top:-3px`（按钮高 12px 实际图标 16px，偏移量不符）导致箭头比文字偏低；改用 `transform: translateY(-50%)` 居中，箭头与文字垂直对齐。
- **下拉框不跟随滚动**：`r-select` 的下拉框挂在 `document.body` 上、仅在打开时定位一次，页面（或滚动容器）滚动时会与触发器脱离（如置于 sticky 头部的语言切换器）。改为打开时监听 `scroll`（capture，可捕获嵌套滚动容器）+ `resize` 重新定位，关闭/卸载时移除监听。
- **浮层 z-index 偏低 / 缺少阶梯**：`r-dropdown` 的 `:host` z-index 只有 `10`（与常见 sticky 头部相同），而真正决定层级的是被消费方定位、portal 到 `<body>` 的 host——内部 `.ranui-dropdown: 1050` 被困在 host 的层叠上下文里，于是下拉/气泡会被页面盖住（消费方只能用 `z-index: 100000` 临时兜底）。新增一套浮层 z-index 阶梯 token：`--ran-z-modal: 1000` < `--ran-z-dropdown: 1100` < `--ran-z-message: 1200`（dropdown 高于 modal，保证 modal 内的 select 可见；toast 永远在最上）。dropdown host / modal / message 改为引用对应 token，并暴露 `--ran-dropdown-host-z-index` 等组件级变量，消费方可调而无需 `!important`。修好后消费方那条 `r-dropdown[role=listbox] { z-index: 100000 }` 兜底即可移除。
- **类型化自定义元素**：`index.ts` 增加 `HTMLElementTagNameMap` 增强，`document.querySelector('r-select')` / `createElement('r-input')` 直接得到组件类型，读取 `.value` 等属性无需 `as` 断言。
- **可发现性**：README 增加 “Components & API reference” 区块，指向自动生成的 `docs/COMPONENTS.md`（含事件 `detail` 形状）与 `docs/DESIGN.md`，并修正过时的「Theme Packs」章节为现行 light/dark token API。
- **外部可定制**：`r-colorpicker` 触发块导出 `::part(block)` / `::part(swatch)`；面板内容因 portal 到 light DOM，可直接用 `.ran-color-picker-*` 类名定制。
- **demo 语言切换器**：改为无边框 + 前置地球图标（新增 `assets/icons/globe.svg`），用 `type="text"` 去边框、`--ran-select-selection-icon-display: none` 隐藏箭头、`currentColor` 让图标与文字同色随 hover 变化。
