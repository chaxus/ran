## packages/ranui

### r-select：下拉箭头图标自注册 + 暗色适配

- **问题**：`r-select` 渲染 `<r-icon name="arrow-down">` 作为下拉箭头，却从未注册该图标，导致箭头缺失并打印 `icon not registered` 警告——除非使用方自己注册 `arrow-down`。箭头颜色还硬编码为 `#d9d9d9`，暗色模式下不跟随主题。
- **修复**：select 在模块加载时 `registerIcon('arrow-down', …)` 自注册图标，组件自包含；箭头颜色改为 `var(--ran-color-text-secondary)`，随主题翻转。

### demo：图标注册时序

- 抽出 `demo/register-icons.ts`，作为第一个 side-effect import，确保 demo 图标在任何组件模块（尤其是会在加载时注册图标的 `r-message`）之前注册，消除路由内容挂载时的 `icon not registered` 警告。

### 可视化回归测试：对齐路由化 demo

- demo 重构为基于 `r-router` 的多页应用后，组件展示从扁平根页移动到 `/components` 路由，原 e2e 视觉用例的 `#component-*` 选择器全部 element-not-found。
- 为 `/components` 路由的每个展示块补充稳定的 `id="component-*"` 锚点；tabs 使用语义化 `r-key`（overview/api/theming/disabled）。
- 用例导航到 `/components`，并将 `networkidle` 改为 `domcontentloaded`（player 会持续拉取 HLS 流，`networkidle` 永不静默会导致导航超时）。

### r-colorpicker：面板重做

- **根因**：`r-popover` 会把面板内容 portal 到 `document.body`，脱离组件 shadow root，于是所有 shadow 作用域 CSS 失效——饱和度方块坍缩为 0px、滑块脱锚浮动、输入框沦为无功能装饰，且大量硬编码颜色不支持暗色。
- **重做**：
  - 面板样式移入 `panel.less`，以 `<style>` 注入面板子树，随 portal 内容一起迁移；选择器唯一命名、全部走 token，暗色自动生效（token 从 light DOM 的 `:root` 继承）。
  - 用基于百分比、与布局无关的自定义 hue/alpha 滑块替换脆弱的 `r-progress` 滑块。
  - 接通取值输入框 + HEX/RGB 格式选择，可显示与编辑颜色，并派发 `change` 事件（`{ value, hex, rgb, rgba, alpha }`）。
  - Geist 对齐的间距 / 圆角 / 预览色块 / 棋盘格（token 驱动），触发块加入 hover 态。
- 已在明暗两色验证；契约测试扩展（alpha 解析、`currentValue`、`change` 事件），视觉 e2e 用例通过。
