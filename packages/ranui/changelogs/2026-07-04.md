# Changelog — 2026-07-04

## packages/ranui

对组件库做了一轮无障碍（a11y）审计与修复，覆盖 DESIGN.md §7 里承诺但此前未落地的
几条原则：live region、表单参与、完整键盘导航、尊重 reduced-motion。

### r-message：toast 变成可朗读的 live region

- **问题**：toast 容器是纯 `<div>`，无 `aria-live`/`role`，读屏用户完全收不到提示。
- **修复**：持久的 `.ranui-message` 容器设为 `aria-live="polite"` + `aria-atomic="false"`
  （在 toast 加入前就存在，新 toast 一 append 就被朗读）；单条 toast `aria-atomic="true"`，
  `type` 为 `error`/`warning` 时升级为 `role="alert"`（assertive 打断），其余 `role="status"`。
- 顺带修一个隐患：原先在 constructor 里 `setAttribute` 违反“自定义元素构造时不得添加
  属性”的规范，移到 `connectedCallback`。

### r-img：新增 alt

- 新增 `alt` 属性/属性访问器，透传到内部 `<img>`；未设置时默认空 `alt=""`（装饰性，
  读屏跳过而非朗读文件名/URL）。

### 表单控件：ElementInternals 表单参与 + 无障碍

- 前提：`<r-form>` 用原生 `new FormData(form)` 收集值，而各控件的原生 `<input>` 在
  shadow DOM 里、FormData 收不到 → 必须 `formAssociated` + `setFormValue`。
- **r-checkbox**：宿主成为唯一可达的 `role="checkbox"`（`aria-checked` + roving
  `tabindex` + Space/Enter 键盘 + `aria-disabled`），内部原生 input 降级为装饰
  （`aria-hidden` + `tabindex=-1`）；`setFormValue`（选中报值、未选报 `null`）。
- **r-input**：`formAssociated` + 值变化时 `setFormValue`；给内部 `<input>` 生成 `id`、
  渲染的 `<label>` 设 `for`（原生关联，读屏报名 + 点击聚焦）。
- **r-select**：`formAssociated`；`value` setter 与用户选中都同步 `setFormValue`。
- 兼容：`setFormValue` 全部可选链调用（jsdom 的 `ElementInternals` 无此方法，真实浏览器
  有），`attachInternals()` 在 constructor 内 try/catch 兜 SSR。真实浏览器已验证
  `new FormData(form).get(name)` 能取到 checkbox 值。

### r-tabs：WAI-ARIA tabs 模式

- 头部行 `role="tablist"`；每个头部（在 r-button 的内层可聚焦元素上，避免嵌套
  `role="button"`）设 `role="tab"` + `aria-selected` + `aria-controls` + roving
  `tabindex`（仅激活项可 Tab 到）；面板设 `role="tabpanel"` + `aria-labelledby`，非激活
  项 `aria-hidden`。
- 键盘：方向键 + Home/End 在标签间移动并自动激活、移焦点。

### r-colorpicker：键盘化

- hue/alpha 滑块：`role="slider"` + `aria-label` + `aria-valuemin/max`，方向键调整
  （Shift 粗调、Home/End 到端点），`aria-valuenow`/`valuetext` 随值实时更新。此前仅鼠标拖拽。
- 触发色块：`role="button"` + `aria-haspopup="dialog"` + `aria-label` + `tabindex`，
  Enter/Space 打开面板（此前键盘完全无法打开取色器）。
- 2D 饱和度/明度面板暂未处理（双轴 slider 较特殊），留待后续。

### 全局：prefers-reduced-motion

- 组件都是 shadow DOM，光 DOM 的全局样式进不去。`ensureShadowRoot` 向每个 shadow root
  的采用样式追加一段 `@media (prefers-reduced-motion: reduce)` 覆盖（排组件 CSS 之后以
  取胜），同时覆盖 constructable stylesheet 与 `<style>` 降级两条路径。一处改动覆盖全库。
