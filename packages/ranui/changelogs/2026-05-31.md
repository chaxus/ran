# Changelog — 2026-05-31

## packages/ranui

### 主题系统与 Demo 优化

#### 主题 token 收敛

- **`theme/tokens.less`** 与 **`theme/dark.less`** — 新增页面级语义 token：
  - `--ran-page-background`
  - `--ran-page-text`
  - `--ran-page-text-muted`
  - `--ran-surface-background`
  - `--ran-surface-background-muted`
  - `--ran-surface-border`
  - `--ran-surface-shadow`
  - `--ran-accent-color`
  - `--ran-accent-color-secondary`
  - `--ran-warning-color`

- **`theme-packs/*.less`** — 为每个主题包补齐页面级 token，让 demo 页面、组件容器和 Shadow DOM 组件使用同一套主题语义。

#### Demo 整体性修复

- **`demo/demo.css`** — 删除 demo 自己按 pack 维护的 `--page` / `--surface` / `--ink` 调色板，改为从 `--ran-*` token 派生。
- **`demo/demo.css`** — 将 `r-card` / `r-section` 桥接到页面级 token，修复深色 theme pack 下卡片标题、描述和嵌套控件仍使用默认组件文字颜色的问题。
- **`demo/index.html`** 与 **`demo/demo.css`** — 将首屏调整为 Theme Workbench：强化实时组件预览、增加轻量网格背景、能力指标和 workbench 面板，让 demo 第一屏更像可操作的组件实验台。
- **`demo/index.html`** 与 **`demo/demo.css`** — 将 Theme Lab 从静态色块改成可点击的 theme micro-window，特殊主题能直接呈现像素、复古系统、手绘、纸张和 brutalism 的视觉语言。
- **`demo/index.ts`** — 显式引入 `../style`，确保默认主题也加载基础 token。
- **`demo/index.ts`** — 新增 `syncThemeControls`，从 `localStorage` 同步导航栏 Theme / Pack 选择器状态。
- **`demo/index.ts`** — 新增 `bindThemePackButtons`，Theme Lab 卡片点击后会同步 `setThemePack()`、导航栏选择器、active 状态和 wired runtime overlay。
- **`demo/index.html`** — Theme 选择器补充 `System` 选项；移除 hero 预览控件上的 `data-wired-skip`，让主题包在首屏组件上完整生效。
- **`test/unit/demo-home.source.test.ts`** 与 **`test/unit/theme-packs.source.test.ts`** — 增加 demo 主题契约和 theme pack 页面级 token 合约测试。

#### Wired Runtime 主题增强

- **`theme-packs/wired-overlay.ts`** — 新增 `syncWiredBordersForThemePack()`，把 `wired` 的 runtime overlay 激活/清理收敛到一个入口。
- **`theme-packs/wired-overlay.ts`** — 新增 `installWiredThemePackSync()`，监听 root `data-ran-theme-pack` 变化，支持“先导入 wired pack、后续再调用 `setThemePack('wired')`”的使用方式。
- **`theme-packs/wired.ts`** 与 **`demo/index.ts`** — 主题入口负责安装 runtime 同步器，demo 切换 pack 时只触发一次同步，避免各自维护 activate/deactivate 逻辑。
- **`theme-packs/wired-overlay.ts`** — rough 边框颜色改为读取 `--ran-skin-outline-color`，让 wired 线条跟随主题 token。
- **`theme-packs/wired-overlay.ts`** — MutationObserver 额外监听 `data-wired-skip`，元素切换为 skip 后会清理已绘制边框。
- **`test/unit/wired-overlay.test.ts`** 与 **`test/unit/theme-packs.source.test.ts`** — 增加 wired runtime 同步、skip 清理和源码契约测试。

### 新增 (Added)

#### Builder 公共子路径导出

- **`builder.ts`** — 新增根级 builder 公共入口，显式导出 `utils/builder` 的公开 API：
  - DOM builder：`View`、`Div`、`Span`、`Slot`、`ButtonBuilder`、`InputBuilder`、`Style`、`Label`、`Ul`、`Li`、`Section`、`Article`、`Nav`、`Header`、`Footer`、`Main`、`DeclarativeShadow`
  - 核心类型与类：`ElementBuilder`、`ShadowBuilder`、`createRef`、`Ref`
  - 事件管理：`EventManager`
  - 响应式原语：`signal`、`createEffect`、`computed`、`batch`、`Getter`、`Setter`、`SignalOptions`
  - SSR / mock 辅助：`isSSR`、`DocumentFragmentMock`、`HTMLElementMock`、`ShadowRootMock`
  - 转义与 mock selector：`escapeHtml`、`escapeHtmlAttribute`、`matchSelector`、`collectMatches`、`MockNode`

- **`package.json`** — 新增 `./builder` package export：

```json
"./builder": {
  "types": "./dist/builder.d.ts",
  "import": "./dist/builder.js",
  "require": "./dist/index.cjs"
}
```

使用方式：

```ts
import { Div, EventManager, signal, createEffect } from 'ranui/builder';
```

- **`vite.config.ts`** — 在 ES 构建入口中新增 `builder: resolve(__dirname, 'builder.ts')`，发布构建会生成：
  - `dist/builder.js`
  - `dist/builder.d.ts`

#### 导出契约测试

- **`test/unit/package-exports.source.test.ts`** — 新增源码级 package export 合约测试：
  - 校验 `builder.ts` 源入口存在
  - 校验 Vite ES 构建包含 `builder` 入口

  - 校验 public entrypoints `index.ts` 与 `builder.ts` 不使用 `export *`

### 变更 (Changed)

#### 根入口去桶导出

- **`index.ts`** — 将根入口的组件与主题 `export *` 改为显式命名导出，降低无意扩展公共 API 的风险。
- 根入口继续保留组件 side-effect imports，用于注册 Web Components。
- `utils/builder` 没有加入根入口；builder API 通过独立子路径 `ranui/builder` 暴露，避免根入口继续膨胀。
