# Changelog — 2026-06-27

## packages/ranui

### 主题系统重做：采用 Geist 设计系统

#### 背景与动机

旧的主题体系维护成本高：7 个 opt-in 主题包（pixel-retro / windows-98 / windows-xp / system-6 / wired / paper / neo-brutalism）+ `--ran-skin-*` 皮肤原语层 + `dark-overrides.less`，暗色覆盖依赖双属性选择器拼优先级，`wired` 包还带运行时 SVG overlay 不支持 SSR。同时 `tokens.less` 与 `dark.less` 存在重复且 hex 值冲突的暗色定义。

本次目标：拆除全部主题包，仅保留最基础的 light/dark，再以业界标准 [Geist 设计系统](https://vercel.com/design) 重新设计 token 体系，并逐层对齐组件。

---

#### 阶段一：拆除主题包

- 删除整个 `theme-packs/` 目录（7 个包 + `dark-overrides` + `transitions` + `wired-overlay`/`wired-assets`）
- 删除 `bin/generate-wired-assets.ts` 与 `roughjs` 依赖
- 移除 `package.json` 中所有 `./theme-packs/*` 导出、`vite.config.ts` 中对应构建入口
- `utils/theme.ts` 精简：移除 `RanThemePackName` / `setThemePack` / `getThemePack` 及 pack 存储逻辑，仅保留 `setTheme` / `getTheme` / `initTheme`（light/dark/system）与 token API（`setThemeToken` / `setThemeTokens` / `clearThemeToken`）
- 删除死文件 `theme/color.less`（无人 import）、`theme/compat.less`（旧别名 `--bg-color`/`--line-color`/`--text-color`/`--brand-color` 等零消费者）
- 删除相关测试：`theme-packs.source` / `wired-overlay`（unit + e2e）/ `theme-packs.spec`

#### 阶段二：Geist token 体系

新的三层 token 架构（`theme/tokens.less` + `theme/dark.less`）：

- **基础调色板**：Geist 全量 scale —— `--ran-gray-100..1000`、`--ran-gray-alpha-100..1000`、`--ran-blue/red/amber/green-100..1000`、`--ran-background-100/200`
- **语义层**：`--ran-color-*` 映射到 base scale，例如
  - `--ran-color-primary: var(--ran-blue-700)`
  - `--ran-color-text: var(--ran-gray-1000)`
  - `--ran-color-bg: var(--ran-background-100)`
  - `--ran-color-border: var(--ran-gray-400)`
  - 新增 `--ran-color-primary-hover/active`、`--ran-color-bg-subtle`
- **暗色单一来源**：`theme/dark.less` 用 LESS mixin `.ran-theme-dark()` 只重定义 base scale 的暗色值；语义 token 通过 `var()` 自动翻转，无需逐个重写。覆盖系统偏好（`@media (prefers-color-scheme: dark)` + `:root:not([data-ran-theme='light'])`）与显式 `[data-ran-theme='dark']` / 旧 `[theme='dark']`
- **其余 Geist token**：radius `sm 6 / md 12 / lg 16 / full`、4px spacing scale（`--ran-space-1..24`）、subtle 阴影（`--ran-shadow-elevated/menu/modal`）、`--ran-focus-ring`、Geist Sans/Mono 字体栈、收紧的 motion 时长
- **skin 层精简**：从 19 个砍到组件实际消费的 4 个（`border-width` / `border-style` / `raised-shadow` / `font-family`），删除全部 pack 残留（`rough-*` / `pixel-size` / `jitter-*` / `surface-texture` / `hard-shadow` / `control-height-*` 等）

#### 阶段三：组件层对齐 Geist

- **button**：圆角 `2px → radius-sm`；primary 的 hover/active 按色阶递进（`blue-700 → 800 → 900`）；新增 `:focus-visible` 焦点环；过渡时长用 `--ran-motion-duration-base`
- **input**：圆角 → `radius-sm`；`:active` 改为正确的 `:focus-within` 焦点态，焦点环用 `--ran-blue-400`（暗色自动翻转）
- **select / dropdown / modal / message**：圆角 token 化（菜单 `radius-md`、modal `radius-md`、内部按钮 `radius-sm`）；dropdown 用 `--ran-shadow-menu`、modal 用 `--ran-shadow-modal`
- **link / checkbox / progress**：补充 `:focus-visible` / `:focus-within` 焦点环
- **tab**：fallback 色对齐 Geist blue

#### 暗色模式 bug 修复

修复了一批在暗色下表现错误的写死颜色，统一改为会随主题翻转的 token：

- **dropdown-item**：黑色文字 `rgba(0,0,0,.88)` → `--ran-color-text`；hover 背景 → `--ran-gray-alpha-100`；选中态 `#e6f7ff` → `--ran-blue-100`
- **skeleton**：底色与微光渐变的 `rgba(0,0,0,…)`（暗色下完全不可见）→ `--ran-gray-alpha-200/500`
- **radar**：canvas 轴标签字色在 JS 中写死黑色 → 新增 `get fontColor()` 读取 `--ran-color-text`；网格线 `#e6e6e6` → `--ran-color-border`

> 核心原则：所有 fallback 指向会随主题翻转的 token（`gray-alpha` / `blue-100` / `color-text`），暗色自动正确，无需为每个组件维护暗色覆盖规则。

### Demo 页面重建

旧 demo（含主题包画廊、i18n、90KB 的 `index.html`）随主题包一并删除。本次重建为一个轻量、token 驱动的 Geist 风格展示页：

- `demo/index.html`：顶栏（品牌 + GitHub/Issues 链接 + 语言切换 + 明暗切换）、hero、色阶展示、radius/elevation 展示、按钮、表单（input/select/checkbox）、反馈（progress/skeleton/loading/message）、表面（card/tabs/modal/link）、数据可视化（radar）
- `demo/demo.css`：页面 chrome 完全由 `--ran-*` token 驱动，切换暗色时整页一起重新着色
- `demo/index.ts`：注册组件与图标、明暗切换、语言切换、从 CSS 变量动态渲染色阶 swatch、注入 radar 数据、modal 开关
- 色阶 swatch 直接以 `background: var(--ran-{scale}-{step})` 渲染，直观演示 token 在明暗模式下的翻转

#### 导航栏 + 多语言

- 顶栏新增 **GitHub**（`github.com/chaxus/ran`）与 **Issues**（`.../issues`）外链，各带一个 `r-icon`（新增 `assets/icons/github.svg` / `issue.svg`，`fill` 走 `currentColor` 随主题自适应）
- demo 字典拆分为 `demo/locales/en.json` + `demo/locales/zh.json`，通过 JSON 导入（`tsconfig` 开启 `resolveJsonModule`）
- 语言切换改用 `r-select`（English / 中文）替代按钮，dogfood 选择器组件；选择持久化到 localStorage，首次访问按 `navigator.language` 自动判定中英文，切换一键翻转整页

#### 新增 i18n 工具：`utils/i18n`（与 router 同级的公共工具）

把多语言能力抽成框架无关的核心，复用 router 的设计（核心引擎 + 全局单例 + 薄 DOM 绑定）：

- `utils/i18n/index.ts` 导出 `I18nCore` / `createI18n(config)` / `useI18n()`，从主 barrel `ranui` 导出，新增 vite 构建入口 `utils/i18n`
- `I18nCore`：`t(key, params)`（缺失回退到 fallbackLocale 再回退到 key 本身，`{param}` 插值）、`setLocale` / `getLocale` / `locale`、`onChange(handler)→unsub`、`addMessages` / `getMessages` / `availableLocales`、`destroy`
- 配置：`locale` / `fallbackLocale` / `messages` / `persist` / `storageKey` / `detectNavigator`；初始 locale 优先级 = 持久化 > 显式 > navigator > fallback
- SSR 安全：`localStorage` / `navigator` 访问均有 guard，构造函数不触碰浏览器全局
- demo 的 `demo/i18n.ts` 重构为消费该核心：`createI18n({ messages:{en,zh}, persist, detectNavigator })`，`applyLanguage` 仅保留 `data-i18n*` 的 DOM 绑定（用 `i18n.t(key)`）
- 新增 `test/unit/utils.i18n.test.ts`（16 个用例：翻译/回退/插值、setLocale 通知与去重、订阅退订、addMessages、持久化、destroy）
