Design system

ranui's tokens follow the Geist design system (design.md / design.dark.md). Base scales → semantic tokens → component tokens; dark mode redefines only the base scale.

Color

Ten-step scales where each step maps to a fixed interface state — color is just how a state looks.

Each step has one job, so hover / active / border / text states are decided up front — no guessing per component:

Spacing

A 4px base unit with just nine values. 8px within a group, 16px between groups, 32–40px between sections — a limited set keeps the rhythm.

Accessibility

The system serves everyone — keyboard users, low vision, bright sunlight.

  • Keep sufficient text-to-background contrast (WCAG AA).
  • Never signal state with color alone — pair it with an icon or text.
  • Every interactive element has a visible focus ring.
  • Full keyboard navigation; nothing is mouse-only.

Light & dark

Semantic tokens reference the base scale, so a single dark mixin redefines the scale and everything re-resolves. Same token name, different value — gray-1000 is #171717 in light, #ededed in dark.

:root[data-ran-theme="dark"] {
  .ran-theme-dark(); /* only the base scale flips */
}
/* --ran-color-text: var(--ran-gray-1000)  →  #ededed */

Guide

Everything you need to install ranui, theme it, localize it, and render it on the server.

Installation

Install the package, then import the global styles once and the components you need.

npm i ranui

import 'ranui/style';   // tokens + light/dark
import 'ranui';         // register all components

Theming API

Switch theme or override any token at runtime. SSR-safe.

import { setTheme, setThemeTokens } from 'ranui';

setTheme('dark');            // 'light' | 'dark' | 'system'
setThemeTokens({
  '--ran-color-primary': '#6c47ff',
  '--ran-radius-md': '10px',
});

Internationalization

A framework-agnostic i18n core (the same one driving this page's language switch).

import { createI18n, useI18n } from 'ranui';

const i18n = createI18n({
  messages: { en, zh },
  fallbackLocale: 'en',
  persist: true,
  detectNavigator: true,
});
useI18n().t('hero.title');
useI18n().setLocale('zh');

Server-side rendering

Render components to a Declarative Shadow DOM string and hydrate on the client.

import { renderToString } from 'ranui/ssr';

const html = renderToString(document.createElement('r-button'));
// → <r-button><template shadowrootmode="open">…</template></r-button>
